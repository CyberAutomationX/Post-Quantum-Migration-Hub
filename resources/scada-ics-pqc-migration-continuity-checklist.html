Recommended use

  1. Start with OT_Crypto_Inventory, using redacted site/zone names where output may be shared publicly.
  2. Capture latency and performance baselines before any PQC or hybrid test.
  3. Use Latency_Test_Cases in a representative lab before production scheduling.
  4. Request vendor roadmaps and rollback guidance for gateways, firmware, remote access, and update mechanisms.
  5. Treat unsupported systems as managed exceptions with compensating controls and lifecycle plans.

Standards and source alignment

Aligned to NIST SP 800-82 Rev. 3's OT safety/performance/reliability emphasis, CISA OT PQC considerations, and NIST supply-chain risk-management guidance.

Core references include NIST FIPS 203, NIST FIPS 204, NIST FIPS 205, NIST NCCoE Migration to PQC, NIST CSWP 39 crypto-agility guidance, CISA quantum-readiness resources, and CISA's 2026 product-category announcement for PQC adoption.
